Detailed Description

The GameObject may be the backbone of Tempest. This is the basic object that will exist in the game. It is the root of all rendered and interactive objects in the games. It is not a manager. This is an abstract class, so it is meant to be full implemented as needed. This holds the generic ideas of what makes up the most basic type of Object in the game Level.

◆ v_Render()

void GameObject::v_Render ( void  )
virtual

Default Render will check if there is a texture attached to the Object, make the shader on the object active, and will send the vertices that have been added to the object to OpenGL using glDrawArrays. This is virtual because it can be overloaded if this is not what you need for rendering.

Reimplemented in Tempest::Glyph.
