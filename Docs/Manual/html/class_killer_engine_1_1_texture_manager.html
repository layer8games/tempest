Detailed Description

The TextureManager is a singleton class that will control when and how all textures are added to memory from the disc. SOIL is the library used to actually load the images from the hard drive and use the image data to create the texture. Once a Texture is created, it can be accessed by ID to copy a pointer to the Texture. This means that each Texture can be shared between multiple objects.
